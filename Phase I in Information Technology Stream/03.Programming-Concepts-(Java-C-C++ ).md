# **1. Programming Concepts (Java / C / C++)**

Programming is the process of writing **instructions for a computer** to perform tasks.

**Key elements:**

* **Variables:** Store data values (e.g., `int a = 5;`)
* **Data Types:**

  * C/C++: `int`, `float`, `double`, `char`, `bool`
  * Java: `int`, `float`, `double`, `char`, `boolean`, `String`
* **Operators:** `+`, `-`, `*`, `/`, `%`, `&&`, `||`, `!`, relational operators
* **Statements:** Instructions executed by the program
* **Syntax:** Rules to write correct programs

**C vs C++ vs Java:**

| Feature             | C                        | C++                               | Java                           |
| ------------------- | ------------------------ | --------------------------------- | ------------------------------ |
| Paradigm            | Procedural               | Procedural + OOP                  | Object-Oriented                |
| Memory Management   | Manual (malloc/free)     | Manual + constructors/destructors | Automatic (Garbage Collection) |
| Platform Dependency | Platform-specific        | Platform-specific                 | Platform-independent (JVM)     |
| Exception Handling  | Limited (setjmp/longjmp) | try/catch                         | try/catch/finally              |

# **2. Program Control (Iteration, Recursion, Functions)**

### **Iteration (Loops)**

Used to execute a block of code multiple times.
**Types:**

* **for loop:** Known number of iterations
* **while loop:** Condition-controlled iteration
* **do-while loop:** Executes at least once

**Example (C++):**

```cpp
for(int i = 0; i < 5; i++) {
    cout << i << " ";
}
```

### **Recursion**

A function calls itself to solve a problem.

* Must have a **base condition** to avoid infinite recursion.
* Used in factorial, Fibonacci, tree traversal.

**Example (Factorial in C++):**

```cpp
int factorial(int n) {
    if(n == 0) return 1;
    return n * factorial(n-1);
}
```

### **Functions**

Reusable blocks of code.

* Defined once, called multiple times.
* Helps **modularize code**.

**Syntax (C++):**

```cpp
int add(int a, int b) {
    return a + b;
}
```

# **3. Scope of Variables**

Defines **where a variable is accessible**.

| Scope            | Description                                  | Example                             |
| ---------------- | -------------------------------------------- | ----------------------------------- |
| Local            | Accessible only inside the block or function | `int x = 5;` inside a function      |
| Global           | Accessible throughout the program            | `int x;` declared outside functions |
| Class (Java/C++) | Accessible as per access modifiers           | `private int age;`                  |

---

# **4. Binding of Variables & Functions**

**Binding:** The process of linking **variables or function calls to memory or code**.

* **Static Binding (Early Binding):**

  * Resolved at **compile time**
  * Example: Function overloading in C++/Java, operator overloading in C++

* **Dynamic Binding (Late Binding):**

  * Resolved at **runtime**
  * Example: Method overriding in Java using **virtual functions** in C++

---

# **5. Parameter Passing**

How arguments are passed to functions.

| Type              | Description                             | Example                   |
| ----------------- | --------------------------------------- | ------------------------- |
| Pass by Value     | Copies value into function parameter    | `void fun(int x)`         |
| Pass by Reference | Passes address; changes reflect outside | `void fun(int &x)` in C++ |
| Pass by Pointer   | Similar to reference, uses pointer      | `void fun(int *x)`        |
| Pass by Object    | Object reference passed in Java         | `void fun(MyClass obj)`   |

# **6. Functional and Logic Programming**

### **Functional Programming**

* Focuses on **pure functions**, avoids state changes
* Examples: Haskell, Java 8+ lambda expressions, C++ STL algorithms
* Key concepts: Immutability, higher-order functions

### **Logic Programming**

* Based on formal logic; specifies **what the program should do** rather than how
* Example: Prolog
* Used in AI and expert systems

# **7. OOP (Object-Oriented Programming) Concepts**

**OOP Principles:**

1. **Class:** Blueprint for objects
2. **Object:** Instance of a class
3. **Encapsulation:** Hiding data and exposing via methods (`private` + `getter/setter`)
4. **Abstraction:** Hiding complex implementation details
5. **Inheritance:** Reusing properties and methods from parent class
6. **Polymorphism:** One interface, multiple implementations (overloading, overriding)

**Example (Java):**

```java
class Vehicle {
    void start() { System.out.println("Vehicle started"); }
}
class Car extends Vehicle {
    void start() { System.out.println("Car started"); }
}
```

---

# **8. Inheritance**

**Types of Inheritance (C++/Java):**

* **Single:** Child inherits from one parent
* **Multilevel:** Grandchild inherits through parent
* **Multiple (C++ only):** Child inherits from multiple parents
* **Hierarchical:** Multiple children from one parent

**Advantages:** Code reusability, extensibility

---

# **9. Constructors**

* Special methods called when an **object is created**
* Used to **initialize objects**
* **Types:**

  * Default constructor: No parameters
  * Parameterized constructor: With parameters
  * Copy constructor (C++): Copies another object

**Example (C++):**

```cpp
class MyClass {
public:
    int x;
    MyClass(int val) { x = val; } // Parameterized constructor
};
```

---

# **10. Functions (Advanced)**

* **Function Overloading:** Same name, different parameters (C++/Java)
* **Function Overriding:** Child class redefines parent method (Java/C++)
* **Recursive Functions:** Call themselves for repeated computation

---

# **11. Exception Handling**

* Mechanism to **handle runtime errors** gracefully
* Prevents program from crashing
* Java/C++ supports **try-catch-finally**

**Example (Java):**

```java
try {
    int a = 10 / 0; // ArithmeticException
} catch(ArithmeticException e) {
    System.out.println("Cannot divide by zero");
} finally {
    System.out.println("Execution completed");
}
```

**Key terms:**

* `try` – Code that might throw exception
* `catch` – Handle exception
* `finally` – Executed regardless of exception
* `throw` – Manually throw exception
* `throws` – Declare checked exceptions
