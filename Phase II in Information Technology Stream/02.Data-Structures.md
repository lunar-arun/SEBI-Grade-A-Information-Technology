# 1Ô∏è‚É£ **ARRAY**

## **Definition**

An **array** is a linear, contiguous data structure that stores elements of the **same data type** at consecutive memory locations.

## **Characteristics**

* Random access using index
* Fixed size
* Contiguous memory allocation

## **Operations & Complexity**

| Operation            | Average Case |
| -------------------- | ------------ |
| Access               | **O(1)**     |
| Search               | **O(n)**     |
| Insert (at position) | **O(n)**     |
| Delete               | **O(n)**     |

## **Advantages**

* Fast indexing
* Easy to traverse

## **Disadvantages**

* Static size
* Costly insert/delete due to shifting

## **Applications**

* Matrices
* Lookup tables
* Implementing heaps, hash tables, etc.

---

# 2Ô∏è‚É£ **LINKED LIST**

## **Definition**

A **linked list** is a linear data structure where elements (nodes) are stored at non-contiguous memory locations and connected using pointers.

### Types:

* **Singly Linked List** (next pointer)
* **Doubly Linked List** (next + prev)
* **Circular Linked List**

## **Node Structure**

```
| data | next |
```

## **Complexity**

| Operation           | Time |
| ------------------- | ---- |
| Traversal           | O(n) |
| Insert at beginning | O(1) |
| Insert at middle    | O(n) |
| Delete node         | O(n) |
| Search              | O(n) |

## **Advantages**

* Dynamic size
* Fast insertions/deletions at head

## **Disadvantages**

* No random access
* More memory (pointer storage)

## **Uses**

* Queues, stacks
* Graph adjacency lists
* Memory management

# 3Ô∏è‚É£ **STACK**

## **Definition**

A **stack** is a LIFO (Last In First Out) data structure.

## **Operations**

* **push()** ‚Äì insert on top
* **pop()** ‚Äì remove from top
* **peek()** ‚Äì get top without removing

## **Complexity**

| Operation | Time     |
| --------- | -------- |
| Push/Pop  | **O(1)** |
| Search    | **O(n)** |

## **Applications**

* Function calls (call stack)
* Undo/Redo operations
* Balanced parentheses
* Expression evaluation (postfix/prefix)

---

# 4Ô∏è‚É£ **QUEUE**

## **Definition**

A **queue** is a FIFO (First In First Out) structure.

## **Operations**

* **enqueue()** ‚Äì insert at rear
* **dequeue()** ‚Äì remove from front

## **Variants**

* Circular queue
* Priority queue
* Deque (double-ended queue)

## **Complexity**

Push/Pop/Enqueue/Dequeue: **O(1)**

## **Applications**

* CPU scheduling
* Buffer management (IO)
* BFS traversal in graphs

# 5Ô∏è‚É£ **BINARY TREE**

## **Definition**

A hierarchical data structure where each node has at most **two children**: left and right.

## **Terminology**

* Root, Parent, Child
* Level, Height
* Leaf node
* Full/Complete/Perfect Binary Trees

## **Traversals**

### 1. **DFS Traversals**

* **Preorder:** Root ‚Üí Left ‚Üí Right
* **Inorder:** Left ‚Üí Root ‚Üí Right
* **Postorder:** Left ‚Üí Right ‚Üí Root

### 2. **BFS Traversal**

* Level Order Traversal (using queue)

## **Complexity**

| Operation              | Time                    |
| ---------------------- | ----------------------- |
| Insert, Delete, Search | **O(n)** (not balanced) |

## **Applications**

* Expression trees
* Compression (Huffman)
* Routing tables

# 6Ô∏è‚É£ **INDEXING**

## **Definition**

Indexing is a data structure technique to access records in **reduced time** without scanning the entire dataset.

## **Types**

1. **Primary Index** ‚Äì on primary key
2. **Secondary Index** ‚Äì on non-key attribute
3. **Dense Index** ‚Äì index entry for *every* record
4. **Sparse Index** ‚Äì entry for *some* records
5. **B-Tree / B+ Tree Indexes** (Databases)

## **Importance in SEBI/Finance**

Efficient indexing is crucial in:

* High-volume trading systems
* Database query optimization
* Real-time analytics

# 7Ô∏è‚É£ **BINARY SEARCH TREE (BST)**

## **Definition**

A binary tree with properties:

* Left subtree contains nodes with keys **less** than root
* Right subtree contains nodes **greater**
* No duplicates (usually)

## **Operations Complexity**

| Operation | Average      | Worst    |
| --------- | ------------ | -------- |
| Search    | **O(log n)** | **O(n)** |
| Insert    | O(log n)     | O(n)     |
| Delete    | O(log n)     | O(n)     |

Worst case happens when tree becomes **skewed**.

## **Balanced BSTs**

* AVL Tree
* Red-Black Tree
* Splay Tree

These guarantee **O(log n)** operations.

# 9Ô∏è‚É£ **HASHING**

## **Definition**

A technique of mapping keys to buckets using a **hash function**.

## **Hash Function**

`index = h(key)`

## **Collision Resolution Techniques**

1. **Open Addressing**

   * Linear Probing
   * Quadratic Probing
   * Double Hashing
2. **Separate Chaining**

   * Linked lists at each index

## **Complexity**

| Operation | Average  | Worst    |
| --------- | -------- | -------- |
| Search    | **O(1)** | **O(n)** |
| Insert    | O(1)     | O(n)     |
| Delete    | O(1)     | O(n)     |

## **Applications**

* Database indexing
* Caches
* Symbol tables
* Blockchain (hash functions)
