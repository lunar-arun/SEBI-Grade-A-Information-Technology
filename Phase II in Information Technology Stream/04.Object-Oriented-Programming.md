# üåü What is Object-Oriented Programming?

OOP is a programming paradigm based on **objects** that contain both **data (attributes)** and **behavior (methods)**.
It models real-world entities and improves modularity, security, scalability, and maintenance.

Four major OOP principles:
‚úî **Abstraction**
‚úî **Encapsulation**
‚úî **Polymorphism**
‚úî **Inheritance**

---

# 1Ô∏è‚É£ **ABSTRACTION**

## ‚úî Definition

Abstraction means **showing only essential features** of an object while hiding unnecessary details.

> *Focus on what an object does, not how it does it.*

Example:

* When you type `print()` in Python, you do not know internal buffer handling.
* ATM: You interact with buttons, not internal circuitry.

## ‚úî In Programming

Achieved via:

* **Abstract classes**
* **Interfaces**
* **Abstract methods**
* **Function declarations without implementation**

### Example (Java-like)

```java
abstract class TradeOrder {
    abstract void validate();
}
```

## ‚úî Benefits

* Reduces complexity
* Enhances code readability
* Allows implementation hiding
* Helps build scalable systems

## ‚úî Real-World / SEBI / Finance Use

* Market data APIs expose abstract interfaces
* Trading systems use abstract ‚ÄúOrder‚Äù/‚ÄúTrade‚Äù classes
* Risk modules expose abstract methods for calculations

# 2Ô∏è‚É£ **ENCAPSULATION**

## ‚úî Definition

Encapsulation is **binding data and methods together**, and **restricting direct access** to some components.

> *It protects data integrity.*

## ‚úî How it is done

* Private variables
* Public methods (‚Äúgetters‚Äù and ‚Äúsetters‚Äù)
* Access modifiers (private, public, protected)

### Example

```java
class BankAccount {
    private double balance;

    public void deposit(double amt) { balance += amt; }
    public double getBalance() { return balance; }
}
```

Here:

* `balance` is protected from unauthorized modification.
* Only methods can modify it.

## ‚úî Benefits

* Data protection
* Prevents unintended access
* Controlled data modification
* Improves maintainability

## ‚úî Financial Use Cases

* Ensuring account balances cannot be modified directly
* Sensitive data like PAN, Aadhaar, transactions stored privately
* Secure modules in trading platforms

# 3Ô∏è‚É£ **POLYMORPHISM**

## ‚úî Definition

Polymorphism means **same function name, different behaviors**, depending on context.

Two major types:

---

## ‚≠ê A. **Compile-time Polymorphism**

(Static Binding / Method Overloading)

### Example

```java
class Order {
    void place() {}
    void place(String type) {}
}
```

* Same method name
* Different parameters
* Selected at compile time

---

## ‚≠ê B. **Run-time Polymorphism**

(Dynamic Binding / Method Overriding)

### Example

```java
class Order {
    void execute() { System.out.println("General order"); }
}

class MarketOrder extends Order {
    void execute() { System.out.println("Market order execute"); }
}
```

Here final method selected at **runtime**.

## ‚úî Benefits of Polymorphism

* Flexible code
* Extensibility
* Reusability
* Cleaner interfaces

## ‚úî Financial Use Examples

* Different order types (market, limit, stop-loss) override a common `execute()` method
* Different risk models override `calculateRisk()`
* Multiple reporting formats implement `generateReport()`

# 4Ô∏è‚É£ **INHERITANCE**

## ‚úî Definition

Inheritance allows a class (**child/derived class**) to **acquire properties** of another class (**parent/base class**).

> *Code reusability + hierarchical classification.*

---

## Types of Inheritance

1. **Single inheritance** ‚Üí One parent, one child
2. **Multilevel inheritance** ‚Üí Parent ‚Üí Child ‚Üí Grandchild
3. **Hierarchical inheritance** ‚Üí One parent, multiple children
4. **Multiple inheritance** (supported in C++, not in Java for classes; interfaces allow it)

---

### Example

```java
class Order { 
    double price;
}

class LimitOrder extends Order {
    double limitPrice;
}
```

## ‚úî Benefits

* Reusability
* Logical organization
* Reduced code redundancy

## ‚úî Disadvantages

* Overuse leads to tightly coupled hierarchy
* Diamond problem (resolved by interfaces/virtual inheritance)
